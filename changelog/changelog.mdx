## v0.7.0 — April 2026

### Session 8 (cont. 5) — v0.8.0 prep: TF-IDF + MediaTool (Apr 12, 2026)

- **MediaGenerationTool** (`core/tools/media_generation_tool.py`) — CrewAI tool wrapper para muapi.ai; feature-flagged (`media_generation_tool: true`); sin deps externas (urllib); 8 tests
- **Semantic avoidance upgrade** (`core/autonomous_daemon.py`) — `_cosine_similarity()` con TF-IDF; `_apply_semantic_avoidance()` usa `overlap>=2 OR cosine>=0.6`; +3 tests
- **SystemPromptBoundary TF-IDF** (`core/governance.py`) — `_tfidf_similarity()` + segunda pasada semántica en `check_system_prompt_boundary()` gateada por `semantic_boundary_check` flag (false por defecto); 8 tests nuevos; 15 existentes sin regresión
- **feature_flags:** `semantic_boundary_check: true` declarado para v0.8.0
- **Test suite:** 4,800 tests · 0 errors · 147 módulos

### Session 8 (cont. 4) — Test suite 100% clean (Apr 12, 2026)

- **Fix: stale `mcp_server/` namespace package** — directorio vacío heredado de `dof_conflux_mcp.py` eliminado envenenaba imports haciendo que Python 3.3+ lo importara como namespace package antes que `core/mcp_server.py`; 27 errores eliminados
- **Suite completamente limpia:** 4,419 tests · 0 errors · 0 failures · 33 skipped — primera vez en la sesión con 0 errores

### Session 8 (cont. 3) — Semantic Avoidance + GraphifyTool (Apr 12, 2026)

- **DaemonMemory Semantic Avoidance** (`core/autonomous_daemon.py`) — `_apply_semantic_avoidance()` detecta overlap de palabras clave entre la acción propuesta y los top-3 patrones de error histórico; si overlap ≥ 2, substituye con acción fallback segura y loguea para audit trail; 7 tests en `test_daemon_memory_avoidance.py`
- **GraphifyTool** (`core/graphify_tool.py`) — CrewAI `BaseTool` que sirve índice semántico del codebase; fallback AST stdlib: escanea `core/*.py`, extrae clases y funciones, filtra por query en <100ms; listo para plugin `m8e/graphify` en v0.8.0; 10 tests en `test_graphify_tool.py`
- **Test suite** grows to 4,419 tests · 146 modules · 0 errors

### Session 8 (cont. 2) — feature_flags_governance + CI fixes (Apr 12, 2026)

- **feature_flags_governance** (`core/governance.py`) — `ConstitutionEnforcer` now checks feature flags before applying each rule; rules in `dof.constitution.yml` can declare `feature_flag:` for staged rollout; 6 tests in `tests/test_governance_feature_flags.py`
- **CI fix** — `tests/test_api.py` + `tests/test_oags_bridge.py` now import `__version__` dynamically instead of hardcoding `"0.6.0"`
- **GitHub Release v0.7.0** — release notes published at github.com/Cyberpaisa/DOF-MESH/releases/tag/v0.7.0
- **Test suite** grows to 4,419 tests · 146 modules · 0 errors
- **Docs** updated: introduction.mdx, installation.mdx, cli-benchmark.mdx, cli-prove.mdx

### Session 8 (cont.) — Governance Gate + v0.7.0 bump (Apr 12, 2026)

- **AutonomousDaemon governance gate** (`core/autonomous_daemon.py`) — `_gate_instruction()` runs injection detection via `SystemPromptBoundary` before any Telegram order executes; BLOCKED_TOOLS also checked against instruction text
- **7 new gate tests** (`tests/test_autonomous_daemon.py`) — normal pass, injection blocked, escalation blocked, never-raises contract
- **v0.7.0 bump** — `dof/__init__.py` + `pyproject.toml` → 0.7.0 · 4,361 tests · 145 modules
- **MANIFEST.in** — PyPI exclusions: `_internal/`, `tests/`, `logs/`, `*.map` (prevents Claude Code–style source leak)
- **DOFFeatureFlags** (`core/feature_flags.py`) — runtime feature flag registry with YAML constitution loading, 3-layer priority (runtime override → YAML → defaults), 12 flags declared for v0.7.0 and v0.8.0 candidates
- **DaemonMemory** (`core/daemon_memory.py`) — indexes `cycles.jsonl` for historical queries: success rate, error patterns, mode distribution, top failing action; wired into `scan_state()` so the daemon avoids repeating its top 174-failure action
- **feature_flags section** in `dof.constitution.yml` — all 12 flags with descriptions, Policy-as-Code for rollout control
