Diapositiva 386

Vision por Computadora (CV)

Aplicaciones principales de la vision por computadora:

  • Seguridad: Reconocimiento de objetivos, incluyendo autenticacion de identidad, filtrado de redes sociales y gestion de seguridad. Se utilizan conjuntos de datos como Labeled Faces in the Wild (LFW) y MS-Celeb-1M. Los algoritmos incluyen redes neuronales convolucionales (CNN), FaceNet y ArcFace.
  • Reconocimiento y clasificacion de imagenes: incluyendo reconocimiento de objetos y analisis de imagenes medicas. Se utilizan conjuntos de datos como ImageNet, Common Objects in Context (COCO) y ChestX-ray14 para entrenar y probar modelos. Se emplean frameworks de aprendizaje profundo como ResNet, VGG y modelos de la serie Inception.
  • Conduccion autonoma: incluyendo reconocimiento de senales de trafico, deteccion de peatones y evasion de obstaculos. KITTI, Cityscapes y Waymo Open Dataset son populares para conduccion autonoma. Estos conjuntos de datos utilizan metodos de deteccion de objetos como YOLO, SSD y Faster R-CNN. Tambien utilizan metodos de percepcion 3D como PointNet y RangeNet++, que trabajan con datos de lidar.
  • Generacion de imagenes

Procesamiento del Lenguaje Natural (NLP):

  • Bot conversacional
  • Analisis de sentimiento
  • Traduccion automatica
  • Sintesis de voz

Diapositiva 387

Aplicaciones de NLP (continuacion)

  • Chatbot: incluyendo asistente de servicio al cliente y altavoz inteligente. El conjunto de datos de entrenamiento puede contener muchos registros de dialogo y entradas de enciclopedia. Los algoritmos comunes son el modelo secuencia-a-secuencia (Seq2Seq), arquitecturas Transformer como BERT y GPT, y metodos de aprendizaje por refuerzo.
  • Analisis de sentimiento: incluyendo analisis de la tendencia de sentimiento de comentarios de usuarios y publicaciones en redes sociales. El conjunto de datos de resenas de peliculas de IMDb y el conjunto de datos de analisis de sentimiento de Twitter son conjuntos de datos de referencia comunmente utilizados. Los algoritmos comunes incluyen el modelo de bolsa de palabras (BOW), TF-IDF y modelos de aprendizaje profundo como TextCNN, LSTM y LSTM bidireccional.
  • Traduccion automatica: traduccion en tiempo real entre diferentes idiomas. El Workshop on Machine Translation (WMT) proporciona multiples conjuntos de datos de pares de idiomas para entrenar modelos de traduccion. El modelo Transformer de Google y sus variantes, como Transformer-XL y arquitecturas basadas en BERT, son los algoritmos predominantes en este campo.
  • Reconocimiento y generacion de voz: incluyendo la conversion de voz a texto o texto a voz. Son la base de los asistentes de voz y los sistemas de traduccion automatica de voz.

Diapositiva 388

Preparacion de Datos (1)

Recopilacion de datos

  • Sistema de recomendacion: Recopilar datos de comportamiento historico de los usuarios, que pueden incluir historial de compras, registros de navegacion y datos de calificacion. Se pueden usar conjuntos de datos publicos, por ejemplo, el conjunto de datos de calificacion de peliculas MovieLens.
  • CV: Obtener datos diversificados de imagenes o videos en campos relacionados para cubrir diversos objetos, escenarios y condiciones que el modelo deberia reconocer. Considerar el uso de conjuntos de datos publicos como ImageNet, COCO o PASCAL VOC como punto de partida.
  • NLP: Usar rastreadores web para obtener texto relacionado de Internet, u obtener texto de conjuntos de datos publicos, libros, noticias y plataformas de redes sociales. Ademas, los cuestionarios son tambien una forma efectiva de recopilar datos especificos.

Diapositiva 389

Preparacion de Datos (2)

Limpieza de datos

Este proceso elimina informacion incorrecta e innecesaria, como imagenes duplicadas, imagenes irrelevantes, elementos de texto duplicados, errores ortograficos y datos de usuario que estan duplicados o son anomalos.

Ingenieria de caracteristicas (Opcional)

Puede ser necesaria la extraccion de caracteristicas utiles de los datos sin procesar segun los requisitos del sistema de recomendacion. Tales caracteristicas incluyen informacion personal del usuario, atributos del objeto y comportamientos del usuario. El proposito de la ingenieria de caracteristicas es crear caracteristicas que puedan ayudar al modelo a comprender mejor los datos, mejorando la precision de la recomendacion.

Actualmente, CV y NLP no requieren esta operacion.

Diapositiva 390

Preparacion de Datos (3)

Etiquetado de datos

Para tareas de aprendizaje supervisado, es necesario etiquetar cada elemento de datos con la etiqueta correcta. Esto puede realizarse manualmente o usando herramientas automatizadas. La calidad y precision del etiquetado son cruciales para el efecto de entrenamiento del modelo.

Division del conjunto de datos

Dividir los datos limpiados y etiquetados en conjuntos de entrenamiento, validacion y prueba. El conjunto de entrenamiento se usa para entrenar el modelo, el conjunto de validacion para ajustar los parametros del modelo y seleccionar el mejor modelo, y el conjunto de prueba para evaluar el rendimiento del modelo.

Preprocesamiento

Preprocesar los datos, como normalizacion y codificacion, para facilitar su procesamiento por el modelo.

Analisis y evaluacion de datos

Despues de crear un conjunto de datos, realizar un analisis exhaustivo de datos para asegurar su calidad, diversidad y equilibrio. Ademas, es necesario evaluar si el conjunto de datos cumple con los objetivos esperados y prepararse para apoyar el entrenamiento del modelo.

Diapositiva 391

Seleccion y Diseno del Modelo

Seleccionar una arquitectura de modelo de aprendizaje profundo adecuada segun el tipo de tarea y las caracteristicas de los datos. Por ejemplo:

  • Tarea de reconocimiento de imagenes: CNN, como VGG, ResNet y YOLO
  • Tarea de procesamiento de texto: red neuronal recurrente (RNN), memoria a largo-corto plazo (LSTM), unidad recurrente con compuerta (GRU), y Transformer (BERT y GPT)
  • Algoritmo de recomendacion basado en redes neuronales profundas y algoritmo de recomendacion basado en mecanismo de atencion

Disenar la jerarquia del modelo, incluyendo la capa de entrada, capa oculta (capa convolucional, capa recurrente, capa de autoatencion, etc.), capa de salida, y capas auxiliares como la posible capa de agrupamiento (pooling) y capa de normalizacion.

Seleccion de hiperparametros

Determinar los hiperparametros del modelo, como la tasa de aprendizaje, tamano del lote (batch size), fuerza de regularizacion, numero de capas, numero de nodos y tasa de dropout.

Diapositiva 392

Entrenamiento del Modelo

  • Propagacion hacia adelante (Forward propagation): Calcula la funcion de perdida, es decir, la diferencia entre la prediccion y la etiqueta real.
  • Retropropagacion (Backward propagation): Optimiza los parametros del modelo basandose en la funcion de perdida, para que la siguiente prediccion sea lo mas cercana posible al valor real.

Propagacion hacia adelante:

z2 = W1*X → a2 = g1(z2) → z3 = W2*a2 → a3 = g2(z3) → C(W) = 1/2 * (y - a3)^2

Retropropagacion:

Se calculan las derivadas parciales de la funcion de costo respecto a cada peso, propagando el error hacia atras a traves de la red para actualizar los parametros.

Diapositiva 393

Evaluacion del Modelo (1)

Metricas de evaluacion para tareas de clasificacion

  • Exactitud (Accuracy): Es una de las metricas de evaluacion mas basicas. Mide la proporcion del numero de muestras correctamente predichas por el modelo respecto al numero total de muestras.
  • Precision: En una tarea de clasificacion, la precision es la proporcion de predicciones positivas verdaderas entre todas las predicciones positivas realizadas por el modelo.
  • Sensibilidad (Recall): Para clasificacion binaria o multiclase, es la proporcion de muestras positivas predichas por el modelo, reflejando la capacidad del modelo para predecir muestras positivas.
  • Puntuacion F1 (F1-Score): Es la media armonica de la precision y la sensibilidad. Combina precision y recall. Un valor F1 mas alto indica que el modelo logra un mejor equilibrio entre precision y sensibilidad.

Diapositiva 394

Evaluacion del Modelo (2)

Metricas de evaluacion para tareas de regresion

  • Error cuadratico medio (MSE): Es el promedio de las diferencias al cuadrado entre el valor predicho y el valor real. Se utiliza para medir el error de prediccion.
  • Raiz del error cuadratico medio (RMSE): Es la raiz cuadrada del MSE. La unidad es la misma que la del valor de observacion original, lo que refleja de manera mas intuitiva la diferencia entre el valor predicho y el valor real.
  • Error absoluto medio (MAE): Es el promedio de las diferencias absolutas entre el valor predicho y el valor real. En comparacion con el MSE, el MAE es mas robusto ante valores atipicos.

Diapositiva 398

Los Modelos Grandes Son Mas Complejos con Mas Parametros

Los modelos grandes frecuentemente tienen miles de millones o incluso mas parametros y pueden ocupar cientos de gigabytes o mas.

  • Los modelos grandes basados en Transformer utilizan el mecanismo de autoatencion, que genera una gran cantidad de parametros. El mecanismo de autoatencion permite al modelo considerar independientemente cada posicion en la secuencia de entrada y capturar flexiblemente dependencias de larga distancia, lo que incrementa aun mas la demanda de parametros del modelo.
  • Para distribuciones de datos complejas y de alta dimensionalidad, mas parametros ayudan al modelo a ajustarse mejor, reducir el sesgo y mejorar el rendimiento del entrenamiento. En particular, al procesar tareas como el lenguaje natural, el modelo necesita suficiente capacidad para comprender y generar diversas estructuras y significados de oraciones.

Por ejemplo, se estima segun la documentacion que GPT-3 tiene aproximadamente 175 mil millones de parametros. LLaMA2 de Meta AI viene en tres tamanos: 7 mil millones (7B), 13 mil millones (13B) y 70 mil millones (70B) de parametros.

Diapositiva 400

Diversidad de los Datos de Entrenamiento de Modelos Grandes

Entrenar modelos grandes requiere conjuntos de datos a gran escala, de alta calidad y multimodales. Estos datos tipicamente necesitan ser recopilados de diversos dominios y fuentes, y pueden ser textos, imagenes, audio y videos.

Pipeline de desarrollo de modelos E2E (extremo a extremo):

Una cadena de herramientas E2E para desarrollo, entrenamiento e inferencia de modelos permite la colaboracion fluida de DataOps, MLOps y DevOps, mejorando la eficiencia de desarrollo en un 50%.

Diapositiva 401

Los Modelos Grandes Tienen Mejores Capacidades

  • Mejor capacidad de aprendizaje: Los modelos grandes tienen mas parametros y una estructura mas compleja, lo que les permite ajustarse mejor a los datos y aprender de manera mas efectiva. Esto les permite procesar conjuntos de datos mas grandes y encontrar patrones mas profundos, logrando un mejor rendimiento en muchas tareas.
  • Mejor capacidad de generalizacion: Los modelos grandes generalizan mejor, obteniendo buen rendimiento con datos nuevos. Esto los hace adaptables para usos del mundo real y menos propensos al sobreajuste.
  • Mayor capacidad de procesamiento multi-tarea: Los modelos grandes pueden manejar multiples tareas a la vez, eliminando la necesidad de modelos separados para cada tarea. Esto mejora la flexibilidad y eficiencia y reduce los costos de mantenimiento.

Diapositiva 402

Proceso de Entrenamiento de LLM

El proceso de entrenamiento de modelos de lenguaje grandes consta de las siguientes fases:

  1. Pre-entrenamiento: >1 billon de tokens. Datos: texto de internet. Tarea: modelado del lenguaje (predecir el siguiente token). Resultado: LLM pre-entrenado.
  2. Ajuste fino supervisado (SFT): 10K-100K. Datos de demostracion (prompt, respuesta). Tarea: modelado del lenguaje (predecir el siguiente token). Resultado: modelo SFT.
  3. Modelo de recompensa: 100K-1M. Datos de comparacion. Tarea: clasificacion (prompt, respuesta_ganadora, respuesta_perdedora). Resultado: modelo de recompensa.
  4. Aprendizaje por refuerzo: 10K-100K. Prompts. Tarea: aprendizaje por refuerzo (generar respuestas que maximicen la recompensa). Resultado: modelo final.

Notas del instructor

Durante el pre-entrenamiento, usamos enormes cantidades de datos de fuentes como paginas web, Wikipedia, libros, GitHub, articulos y sitios de preguntas y respuestas. Estos datos contienen de cientos de miles de millones a billones de palabras. Entrenamos el modelo usando una supercomputadora con miles de GPUs potentes y redes rapidas. Se necesitan decenas de dias para completar este entrenamiento y construir el modelo base de lenguaje. El modelo base desarrolla la capacidad de modelar textos largos, permitiendole generar lenguaje. Basandose en el prompt de entrada, el modelo puede producir completaciones de texto.

Diapositiva 403

Conjuntos de Datos de Pre-entrenamiento

Las fuentes de datos de los modelos grandes son diversas, e incluyen principalmente las siguientes:

  • Datos publicos en Internet: Internet contiene enormes cantidades de datos publicos, como textos, imagenes y videos. Los desarrolladores pueden rastrear estos datos para construir sus conjuntos de datos. Tales datos pueden provenir de fuentes como sitios de noticias, redes sociales, blogs y foros.
  • Bases de datos profesionales y datos publicados por instituciones: Muchas instituciones profesionales, bases de datos y organizaciones academicas publican conjuntos de datos organizados y procesados. Estos conjuntos de datos suelen tener alta calidad y fiabilidad.
  • Datos internos de la empresa: Para algunos modelos grandes personalizados, las empresas pueden usar sus datos internos para construir conjuntos de datos, incluyendo datos de negocio, comportamiento de usuario y datos de productos.
  • Contenido generado por usuarios: Con la popularidad de las redes sociales y plataformas en linea, el contenido generado por usuarios (como comentarios, calificaciones y retroalimentacion) se ha convertido en una fuente importante de conjuntos de datos para modelos grandes.

Diapositiva 404

Paralelismo de Datos

Paralelismo de datos: El conjunto de datos de entrenamiento se divide en N partes, cada una ubicada en un nodo de computo separado. Cada nodo mantiene una copia completa del modelo y calcula los gradientes usando sus datos locales. Todos los nodos envian sus gradientes al nodo 0, que los combina y envia el resultado final de vuelta a los otros nodos.

Diapositiva 405

Paralelismo de Tensores

En el entrenamiento con paralelismo de tensores, dividimos un tensor en N bloques a lo largo de una dimension. Cada dispositivo contiene 1/N del tensor, manteniendo la computacion precisa. Esto requiere comunicacion adicional para asegurar resultados correctos.

Notas del instructor

  • Particion de pesos por filas: El numero de unidades de computo determina como se dividen los parametros. Los datos tambien necesitan dividirse para los calculos matriciales.
  • Particion de pesos por columnas: El numero de unidades de computo determina como se divide w. En este caso, los datos de entrada X no necesitan dividirse.

Diapositiva 406

Paralelismo de Pipeline

La siguiente figura muestra el paralelismo de pipeline. Supongamos que hay cuatro NPUs. Los parametros del modelo grande se dividen en cuatro partes por capa y se colocan en los dispositivos 0, 1, 2 y 3 en secuencia.

Para un minilote de datos, primero se realiza la propagacion hacia adelante en el dispositivo 0 (correspondiente al bloque gris F0). Despues de completar el calculo, el resultado del dispositivo 0 se envia al dispositivo 1 para continuar la propagacion hacia adelante (bloque amarillo F0). Despues de completar la propagacion hacia adelante en la ultima NPU (dispositivo 3), se calcula la perdida en el dispositivo 3 y se realiza la retropropagacion (bloque purpura B0). La retropropagacion se realiza hasta completar en el dispositivo 0. Los parametros del optimizador se actualizan en cada tarjeta y se actualizan los parametros del modelo.

Diapositiva 408

Ajuste Fino Supervisado (SFT)

  • SFT se refiere a pre-entrenar un modelo de red neuronal (el modelo fuente) en un conjunto de datos fuente. Luego se crea un nuevo modelo de red neuronal (el modelo objetivo).
  • El modelo objetivo copia todos los elementos de diseno y parametros del modelo fuente, excepto la capa de salida.
  • Estos parametros contienen conocimiento del conjunto de datos fuente, que funciona bien con el conjunto de datos objetivo. La capa de salida del modelo fuente se elimina porque esta estrechamente vinculada a las etiquetas del conjunto de datos fuente.
  • Durante el ajuste fino, se agrega una capa de salida cuyo tamano de salida es igual al numero de categorias del conjunto de datos objetivo al modelo objetivo, y se inicializan aleatoriamente los parametros de esta capa. Entrenamos el modelo objetivo desde cero hasta la capa de salida. Los parametros de las demas capas se ajustan finamente usando los parametros del modelo fuente.

Diapositiva 409

Ajuste con Adaptadores (Adapter Tuning)

A medida que el hardware mejora, los modelos pre-entrenados tienen mas parametros. Ajustar finamente todo el modelo para tareas especificas es costoso y lleva mucho tiempo. Los adaptadores ayudan a resolver este problema.

El adapter agrega parametros para tareas especificas en cada capa del modelo pre-entrenado. Durante el ajuste fino, solo se entrenan estos parametros especificos de la tarea, y el modelo principal se congela. Esto reduce los costos de poder de computo.

Se agrega un modulo adaptador a cada capa o algunas capas del modelo pre-entrenado. Durante el ajuste fino, la parte principal del modelo pre-entrenado se congela y el modulo adaptador aprende el conocimiento de la tarea especifica.

Diapositiva 410

Ajuste de Prefijo (Prefix Tuning)

En el ajuste de prefijo, se agrega un conjunto de tokens virtuales especificos de la tarea antes de la entrada como prefijo. Durante el entrenamiento, solo se actualizan los parametros de estos tokens de prefijo, mientras que el resto de los parametros del Transformer permanecen fijos.

Diapositiva 411

Ajuste de Prompts (Prompt Tuning)

Este metodo es una forma mas simple del ajuste de prefijo. Agrega tokens de prompt solo a la capa de entrada, eliminando la necesidad de un MLP para manejar problemas de entrenamiento.

Los parametros pre-entrenados permanecen fijos, y se agregan uno o mas embeddings para cada tarea. Estos embeddings se combinan con la consulta y se alimentan al LLM, entrenandose solo los embeddings.

La figura izquierda muestra el ajuste fino de parametros completos para una sola tarea, mientras que la figura derecha muestra el ajuste de prompts.

Diapositiva 412

LoRA: Adaptacion de Bajo Rango de Modelos de Lenguaje Grandes

La idea de LoRA es simple. Los parametros de la matriz del modelo pre-entrenado se congelan y se reemplazan con las matrices A y B. Durante las tareas especificas, solo se actualizan A y B.

  • Se agrega un bypass junto al modelo de lenguaje pre-entrenado (PLM) para reducir y luego aumentar dimensiones, imitando el rango intrinseco.
  • Durante el entrenamiento, solo se actualizan las matrices de reduccion de dimensionalidad (A) y de aumento (B).
  • El tamano de entrada y salida del modelo permanece igual. Al final, los parametros de BA y del PLM se fusionan.
  • La matriz A comienza con valores gaussianos aleatorios, y la matriz B comienza como una matriz de ceros para mantener la matriz de bypass en cero inicialmente.

Diapositiva 414

Alineacion

Usar datos que los humanos prefieren para el entrenamiento (SFT + RLHF):

  • Autenticidad: ¿Es informacion falsa o enganosa?
  • Inocuidad: ¿Causa dano fisico o mental a las personas o al medio ambiente?
  • Utilidad: ¿Aborda la tarea del usuario?

Ejemplo: ¿Que hacer cuando tu amigo esta molesto?

Ajuste fino supervisado con instrucciones:

  • Disculparse con ellos
  • Discutir con ellos
  • Ir a caminar con ellos
  • Razonar con ellos
  • Recordarles que se cuiden
  • Admitir el error rapidamente

Ajuste fino con aprendizaje por refuerzo: Puntuaciones: -5, -1, +3...

Notas del instructor

  • ¿Por que usar ajuste con instrucciones? Porque las salidas de GPT pueden ser aleatorias, imprecisas, poco interesantes o daninas debido a su entrenamiento con grandes conjuntos de datos.
  • ¿Por que usar aprendizaje por refuerzo? 1. Etiquetar datos es costoso. 2. El aprendizaje por refuerzo muestra si un modelo funciona bien pero no explica como mejorarlo. El aprendizaje por refuerzo permite que los modelos exploren mas libremente, superando los limites del aprendizaje supervisado.

Diapositiva 415

RLHF (Aprendizaje por Refuerzo con Retroalimentacion Humana)

El aprendizaje por refuerzo con retroalimentacion humana (RLHF) es un metodo que aprovecha la retroalimentacion humana para refinar un modelo, permitiendo un autoaprendizaje efectivo.

Al integrar la retroalimentacion humana en la funcion de recompensa, RLHF asegura que los modelos grandes puedan realizar tareas mas alineadas con los objetivos, aspiraciones y necesidades humanas.

Notas del instructor

RLHF es una tecnologia estandar utilizada para asegurar que los LLMs creen contenido real, inofensivo y util. Sin embargo, la comunicacion humana es un acto personal y creativo, y el valor de la salida del LLM depende en gran medida de los valores y preferencias humanas. Diferentes modelos se entrenan de diversas maneras y usan diferente retroalimentacion humana, lo que lleva a resultados variados. Que tan bien cada modelo se alinea con los valores humanos depende de su creador.

Diapositiva 416

Entrenamiento del Modelo de Recompensa (RM)

Se seleccionan preguntas aleatoriamente del conjunto de datos. Se usa el modelo generado despues del ajuste fino supervisado con instrucciones (modelo SFT), y se generan multiples respuestas para cada pregunta. El personal de etiquetado clasifica las respuestas segun una evaluacion exhaustiva. Este proceso es como recibir orientacion de un mentor o profesor.

Luego, se usan los resultados de clasificacion para entrenar el modelo de recompensa. Multiples resultados de clasificacion se emparejan para formar multiples pares de datos de entrenamiento.

El modelo RM toma una entrada y da una puntuacion para calificar la calidad de la respuesta. Para cada par de entrenamiento, ajusta los parametros para asegurar que las respuestas de alta calidad obtengan puntuaciones mas altas que las de baja calidad.

Ejemplo: ¿Que es un platano? → Se generan 4 respuestas (A, B, C, D) → Ordenamiento manual → Se usa el resultado para entrenar el modelo de recompensa.

Notas del instructor

  • Preparacion de datos: Preparar un lote de datos para entrenar el modelo de recompensa, incluyendo muestras de texto y sus puntuaciones de evaluacion humana.
  • Construccion del modelo: Construir una red neuronal como modelo de recompensa. Se puede usar un perceptron multicapa (MLP) o un modelo mas complejo como el Transformer.
  • Proceso de entrenamiento: Usar los datos preparados para entrenar el modelo de recompensa. El objetivo es minimizar el error entre las puntuaciones predichas y las puntuaciones reales de evaluacion humana.
  • Validacion y ajuste: Validar el modelo durante el entrenamiento para asegurar que pueda predecir con precision las evaluaciones humanas en datos nuevos.

Diapositiva 417

Aprendizaje por Refuerzo

  • El aprendizaje por refuerzo considera el impacto general mas que el aprendizaje supervisado. El aprendizaje supervisado da retroalimentacion sobre tokens individuales. El aprendizaje por refuerzo da retroalimentacion sobre toda la salida en lugar de tokens individuales. El aprendizaje por refuerzo funciona mejor para modelos de lenguaje grandes debido a la diferencia en la granularidad de la retroalimentacion.
  • El aprendizaje por refuerzo es mas efectivo para abordar problemas de alucinacion. En el aprendizaje supervisado, el modelo da un resultado incluso si no conoce la respuesta. En el aprendizaje por refuerzo, el modelo evita respuestas de baja puntuacion usando puntuaciones de recompensa.
  • El aprendizaje por refuerzo maneja mejor los problemas de acumulacion de recompensas en conversaciones de multiples turnos. Construir interacciones de dialogo efectivas de multiples turnos es dificil con aprendizaje supervisado. Pero con aprendizaje por refuerzo, podemos crear una funcion de recompensa que evalua la salida del modelo basandose en la coherencia y contexto del dialogo.

Diapositiva 418

Procedimiento de Entrenamiento RLHF

Paso 1: Aprendizaje supervisado

Ajustar finamente el LLM pre-entrenado en temas especificos o con instrucciones y corpus de ejemplos humanos.

  1. Desarrollar manualmente el plan de entrenamiento (que preguntas hacer).
  2. Personal de etiquetado profesional da respuestas de alta calidad a estas preguntas.
  3. Usar estas respuestas para refinar GPT-3 y ajustar finamente el modelo.

Paso 2: Modelo de recompensa

Recopilar conjuntos de datos etiquetados por humanos para entrenar un modelo de recompensa.

  1. Hacer las mismas preguntas al modelo entrenado otra vez. GPT genera multiples respuestas para cada pregunta.
  2. Ordenar y puntuar manualmente los resultados.
  3. Devolver los resultados nuevamente para generar un modelo de recompensa.

Paso 3: Aprendizaje por refuerzo

Usar aprendizaje por refuerzo, como PPO, para ajustar finamente el LLM con este conjunto de datos y modelo de recompensa.

  1. Hacer nuevas preguntas aleatorias.
  2. El modelo entrenado da respuestas.
  3. Usar el modelo de recompensa para clasificar y puntuar las respuestas de salida y entrenar repetidamente el modelo GPT.

Diapositiva 420

¿Por Que Necesitamos Evaluacion para Modelos Grandes?

Un modelo grande es una funcion f altamente capaz, que no es fundamentalmente diferente de los modelos anteriores de aprendizaje automatico. Entonces, ¿por que necesitamos evaluacion para modelos grandes?

  • La investigacion y evaluacion nos ayudan a comprender mejor las fortalezas y debilidades de los modelos grandes. La mayoria de los estudios muestran que estos modelos igualan o superan el rendimiento humano en muchas tareas. Sin embargo, algunos estudios dudan si este exito proviene de memorizar los datos de entrenamiento.
  • La investigacion y evaluacion pueden guiar la mejora de la colaboracion entre humanos y modelos grandes. Dado que las personas son los principales usuarios, comprender y evaluar las capacidades del modelo es crucial para mejorar los disenos de interaccion humano-maquina.
  • La investigacion y evaluacion pueden planificar y coordinar mejor el desarrollo futuro de los modelos grandes y prevenir riesgos desconocidos y potenciales.

Notas del instructor

  • Por ejemplo, cuando se le da solo el numero del problema de LeetCode, el modelo produce la solucion correcta. Esto muestra que los datos de entrenamiento estaban contaminados.
  • Por ejemplo, nuestro estudio reciente, PromptBench, es el primer benchmark de evaluacion de robustez de prompts en LLMs. Encontramos que estos modelos a menudo luchan con la interferencia y carecen de estabilidad.
  • Los modelos grandes siguen evolucionando y volviendose mas poderosos. ¿Podemos evaluar sus habilidades desde un punto de vista evolutivo? ¿Como podemos predecir riesgos potenciales por adelantado?

Diapositiva 421

Dimensiones de Evaluacion

Los modelos de lenguaje grandes pueden manejar muchas tareas complejas de lenguaje natural, a diferencia de los algoritmos tradicionales. Por lo tanto, la evaluacion del modelo debe realizarse desde multiples dimensiones, clasificadas en tres aspectos:

  • Conocimiento y capacidad: Los LLMs tienen conocimiento rico y pueden manejar muchas tareas, incluyendo procesamiento del lenguaje natural (clasificacion de texto y extraccion de informacion), preguntas y respuestas de conocimiento (comprension lectora y Q&A de dominio abierto), generacion de lenguaje natural (resumen de texto y creacion de texto), razonamiento logico y generacion de codigo.
  • Etica y seguridad: Los modelos se entrenan basandose en los principios 3H: utilidad (helpfulness), honestidad (honesty) e inocuidad (harmlessness). La inocuidad asegura que las respuestas del modelo se alineen con los valores humanos.
  • Dominios verticales: principalmente cubriendo capacidad de razonamiento complejo (como razonamiento de conocimiento y razonamiento matematico), capacidad de interaccion con el entorno (por ejemplo, un agente genera acciones basadas en instrucciones de lenguaje para tareas del hogar), y capacidad de resolucion de problemas en dominios especificos (como finanzas, derecho y salud).

Diapositiva 422

Sistema de Evaluacion

  • GLUE y SuperGLUE: Evaluan la comprension del lenguaje natural del modelo. Cubren tareas como clasificacion de texto, inferencia, respuesta a preguntas y analisis de sentimiento. Estos benchmarks simulan escenarios reales de procesamiento del lenguaje.
  • GSM8K y MMLU: Evaluan 57 disciplinas diferentes, proporcionando una evaluacion integral del dominio de conocimiento de los modelos grandes. GSM8K evalua habilidades de resolucion de problemas matematicos. MMLU usa preguntas de opcion multiple para evaluar la comprension y razonamiento del modelo en muchos campos.
  • BLUE-bench: Se enfoca en el campo biomedico y evalua que tan bien los modelos funcionan en Q&A medico y tareas relacionadas.
  • Benchmarks de Robustez Adversarial: Evaluan que tan bien el modelo resiste entradas maliciosas, evaluando su seguridad.
  • Benchmarks de Etica de LLM: Verifican si los modelos grandes crean contenido que rompe la etica social, como sesgo, toxicidad o deshonestidad.

Diapositiva 423

Metricas de Evaluacion Comunes

  • BLEU: Se usa comunmente en tareas de traduccion automatica. Verifica cuantos n-gramas coinciden entre el texto traducido y el texto de referencia. Mas coincidencias significan mejor calidad de traduccion. Los unigramas miden la precision de la traduccion a nivel de palabra, mientras que los n-gramas de orden superior evaluan la fluidez de la traduccion a nivel de oracion.
  • NIST: Es una mejora del metodo BLEU, introduciendo el concepto de volumen de informacion para cada n-grama, dando mas peso a palabras menos frecuentes pero importantes.
  • ROUGE: Es una version mejorada de BLEU que se enfoca en el recall (sensibilidad), no en la precision. Verifica cuantos n-gramas de la traduccion de referencia aparecen en la salida. ROUGE-N y ROUGE-L son puntuaciones basadas en n-gramas y la subsecuencia comun mas larga, respectivamente.

Diapositiva 424

Desafios en la Evaluacion de Modelos Grandes (1)

  • Las diversas capacidades de los modelos fundacionales hacen que la evaluacion sea desafiante: Es dificil hacer evaluaciones completas y justas sobre las diversas capacidades de los modelos fundacionales. Por ejemplo, Stanford evalua modelos en 42 tareas de escenario en 7 dimensiones, SuperCLUE define 70 tareas de escenario, C-Eval selecciona 52 tareas de escenario, y AGIEval elige 20 tareas de examenes estandarizados.
  • Falta de estandares industriales unificados para la evaluacion de modelos grandes: Los estandares y metodos de prueba carecen de uniformidad en toda la industria; diferentes organizaciones de evaluacion tienen diferentes sujetos, alcances y resultados de prueba. La prueba basada en conjuntos de datos publicos presenta desafios significativos: la disponibilidad de numerosos conjuntos de datos publicos aumenta la complejidad de seleccion, y es dificil asegurar que estos conjuntos de datos no se superpongan con el conjunto de entrenamiento del modelo.

Diapositiva 425

Desafios en la Evaluacion de Modelos Grandes (2)

  • Falta de estandares de evaluacion para aplicaciones empresariales: La industria carece de una forma estandar de evaluar aplicaciones empresariales. Diferentes areas se enfocan en diferentes caracteristicas. Por ejemplo, el servicio al cliente valora el cumplimiento de instrucciones y la supresion de alucinaciones, mientras que I+D se enfoca en la generacion de codigo. Los datos de internet de codigo abierto no pueden usarse para evaluar el rendimiento del modelo en servicios empresariales. Las plataformas de prueba profesionales pueden filtrar datos empresariales.
  • Actualizaciones rapidas del modelo, carga pesada de evaluacion manual y trabajo repetitivo: Los modelos se actualizan rapidamente. Las evaluaciones manuales son lentas, repetitivas e inconsistentes. La evaluacion profesional necesita mucha preparacion de datos. El etiquetado por expertos consume mucho tiempo y esfuerzo.

Diapositiva 430

Prompting de Cero Ejemplos (Zero-Shot Prompting)

El prompting de cero ejemplos es una tecnica donde el usuario no proporciona demostraciones relacionadas con el resultado de la tarea, y directamente le pide al modelo de lenguaje que proporcione respuestas relacionadas con la tarea.

Los modelos de lenguaje grandes son capaces de prompting de cero ejemplos, pero su exito depende de la complejidad de la tarea y del alcance del conocimiento existente.

Formato:

P: <Pregunta>?
R:

Diapositiva 431

Ejemplo de Prompting de Cero Ejemplos

Prompt:

"Ciertas sustancias en las manzanas pueden causar alergias en los consumidores, provocando sintomas como erupciones cutaneas y dificultad para respirar; al mismo tiempo, el exceso de fructosa y fibra dietetica en las manzanas puede causar sintomas digestivos como hinchazon y diarrea."

Salida:

Evaluacion positiva y negativa: La descripcion incluye posibles impactos negativos de las manzanas sin mencionar directamente aspectos positivos. Analisis detallado:

  1. Evaluacion negativa: La descripcion senala riesgos potenciales para la salud, detalla sintomas de reacciones alergicas y enfatiza posibles efectos secundarios del consumo de manzanas.
  2. Evaluacion positiva: Aunque no hay evaluacion positiva directa, se puede inferir que las manzanas son ampliamente reconocidas como un alimento saludable.

En resumen, la descripcion se centra en los riesgos potenciales sin expresar caracteristicas positivas, por lo que se juzga como evaluacion negativa.

Notas del instructor

  • Si solo se requiere un juicio simple de datos, la salida es compleja y contiene una gran cantidad de informacion innecesaria.
  • Los modelos de la serie GPT son todos modelos de lenguaje autorregresivos: predicen la siguiente palabra basandose en la entrada actual, luego combinan el resultado de la prediccion con la entrada para predecir la siguiente palabra. Este proceso se repite ciclicamente.
  • Los investigadores de GPT-3 encontraron que los datos de entrenamiento a gran escala generan un comportamiento emergente interesante llamado Aprendizaje en Contexto (ICL). ICL no requiere ajuste de parametros del modelo y puede lograr excelentes resultados con solo unos pocos ejemplos de tareas especificas.
  • El modelo GPT-3 pre-entrenado no necesita ser reentrenado cuando se migra a una nueva tarea. Solo se necesita proporcionar una descripcion de la tarea (opcional), luego varios ejemplos (consultas de tarea y respuestas correspondientes, organizados en pares), y finalmente agregar la consulta a ser respondida por el modelo.

Diapositiva 432

Prompting de Pocos Ejemplos (Few-Shot Prompting)

Aunque los modelos de lenguaje grandes demuestran impresionantes capacidades de cero ejemplos, aun tienen un rendimiento deficiente en tareas mas complejas con configuraciones de cero ejemplos.

El prompting de pocos ejemplos puede usarse como tecnica para habilitar el aprendizaje contextual, proporcionando demostraciones en el prompt para guiar al modelo a lograr un mejor rendimiento. Las demostraciones son condiciones para los ejemplos subsiguientes, y se espera que el modelo genere respuestas.

Los ejemplos (few) pueden seleccionarse aleatoriamente.

Formato:

P: <Pregunta>?  R: <Respuesta>
P: <Pregunta>?  R: <Respuesta>
P: <Pregunta>?  R: <Respuesta>
P: <Pregunta>?  R:

Notas del instructor

One-shot es un caso especial de few-shot.

Diapositiva 435

Prompting de Cadena de Pensamiento (Chain-of-Thought)

El prompting de cadena de pensamiento (CoT) permite el razonamiento complejo al introducir pasos intermedios de razonamiento. Expande una pregunta paso a paso y retiene la informacion de contexto de preguntas anteriores en cada respuesta. CoT hace que una conversacion sea rica en contenido y coherente.

Tipos principales de CoT:

  • Few-shot-CoT (CoT de pocos ejemplos)
  • Zero-shot-CoT (CoT de cero ejemplos)

Caracteristicas:

  • Orden: El prompting CoT requiere que las preguntas se descompongan en una serie de pasos ordenados. Cada paso se basa en el anterior, formando una cadena de pensamiento clara.
  • Relevancia: Debe haber una conexion logica estrecha entre cada paso de pensamiento para asegurar la coherencia y consistencia de todo el proceso de pensamiento.
  • Inferencia paso a paso: El modelo se enfoca solo en la pregunta actual e informacion relacionada en cada paso y se acerca a la respuesta final a traves del razonamiento paso a paso.

Diapositiva 436

Few-Shot-CoT (CoT de Pocos Ejemplos)

En este modo, se proporcionan al modelo un pequeno numero de ejemplos, cada uno de los cuales incluye un proceso de inferencia de alta calidad (escrito a mano o generado por el modelo).

Prompt:

La suma de los numeros impares en este grupo es par: 4, 8, 9, 15, 12, 2, 1.
R: Sumar todos los numeros impares (9, 15, 1) da 25. La respuesta es Falso.

La suma de los numeros impares en este grupo es par: 17, 10, 19, 4, 8, 12, 24.
R: Sumar todos los numeros impares (17, 19) da 36. La respuesta es Verdadero.

La suma de los numeros impares en este grupo es par: 16, 11, 14, 4, 8, 13, 24.
R: Sumar todos los numeros impares (11, 13) da 24. La respuesta es Verdadero.

La suma de los numeros impares en este grupo es par: 17, 9, 10, 12, 13, 4, 2.
R: Sumar todos los numeros impares (17, 9, 13) da 39. La respuesta es Falso.

La suma de los numeros impares en este grupo es par: 15, 32, 5, 13, 82, 7, 1.
R:

Salida: Sumar todos los numeros impares (15, 5, 13, 7, 1) da 41. La respuesta es Falso.

Diapositiva 437

Zero-Shot-CoT (CoT de Cero Ejemplos)

El metodo zero-shot-CoT agrega el prompt "Pensemos paso a paso" antes de cada respuesta del modelo de lenguaje para guiar al modelo a pensar paso a paso.

Diapositiva 438

Ventajas de Zero-Shot-CoT

  • Generalizacion: Zero-shot-CoT usa una sola plantilla y puede funcionar en multiples tareas de inferencia sin necesidad de disenar un prompt especifico para cada tarea, mientras que los prompts de pocos ejemplos requieren un ejemplo dedicado para cada tarea.
  • Sin diseno manual de ejemplos: Zero-shot-CoT no requiere el diseno manual de ejemplos de inferencia de multiples pasos, reduciendo la carga de trabajo de la ingenieria de prompts.
  • Benchmark de inferencia zero-shot mas fuerte: Zero-shot-CoT sirve como un benchmark de inferencia zero-shot mas fuerte, proporcionando mejor rendimiento para tareas dificiles de inferencia de multiples pasos.
  • Mejor escalabilidad: A medida que aumenta la escala del modelo, la mejora de rendimiento de zero-shot-CoT es mas obvia, mostrando mejor escalabilidad. Los prompts de pocos ejemplos no son muy sensibles a la escala del modelo.
  • Sesgo reducido: Zero-shot-CoT reduce el sesgo introducido por los ejemplos disenados manualmente, haciendo que la investigacion sobre el sesgo del modelo de lenguaje sea mas justa.

Diapositiva 460

Recomendacion Inteligente

La recomendacion inteligente es un servicio de sistema de recomendacion construido sobre IA. La recomendacion impulsada por IA puede integrar automaticamente big data y obtener continuamente informacion sobre los usuarios finales a traves del analisis y filtrado de datos, empoderando a las empresas para desarrollarse en organizaciones de toma de decisiones basadas en datos.

Los grafos de conocimiento son bases de conocimiento utilizadas en sistemas de recomendacion inteligente. Son esencialmente redes estructuradas que almacenan relaciones entre entidades. Contienen una gran cantidad de informacion de fondo sobre objetos objetivo y relaciones entre objetos en sistemas de recomendacion. Los grafos de conocimiento proporcionan perfiles precisos de objetos objetivo, como comportamiento del usuario, intereses y requisitos, para implementar coincidencia precisa y recomendacion dirigida, logrando recomendacion personalizada especifica por escenario y tarea.

Diapositiva 461

Caso de Estudio

Como se muestra en la figura, un grafo de conocimiento proporciona diferentes tipos de enlaces de relacion, introduciendo mas relaciones semanticas a los elementos. Esto ayuda a diversificar los resultados de recomendacion, evitar limitaciones de recomendacion y descubrir intereses del usuario a un nivel mas profundo.

Ejemplo: Un usuario al que le gusta "Adios a Mi Concubina" (pelicula con Leslie Cheung como actor principal, dirigida por Chen Kaige, genero: pelicula historica) tambien podria gustarle: "Dias de ser Salvaje", "El Ultimo Emperador" y "Atrapados en la Red".

Ademas, el grafo puede conectar los registros historicos de los usuarios y los resultados de recomendacion para mejorar su satisfaccion y aceptacion de los resultados de recomendacion y aumentar su confianza en el sistema de recomendacion.

Notas del instructor

Ademas, como un tipo de informacion auxiliar efectiva en sistemas de recomendacion hibridos, los grafos de conocimiento pueden resolver eficazmente una serie de problemas clave relacionados con la escasez de datos, el arranque en frio y la diversidad de recomendaciones.

Diapositiva 464

Los Robots Entienden Instrucciones y Cosas

El modelo Vision-Lenguaje-Accion (VLA) puede mejorar significativamente la capacidad de generalizacion de un robot para nuevos objetos. Puede interpretar comandos que no existen en los datos de entrenamiento del robot, por ejemplo, colocar un objeto en un numero o icono especificado. Tambien puede realizar inferencia basica en respuesta a comandos del usuario, como recoger el objeto mas pequeno o mas grande, o el mas cercano a un objeto especificado.

Los modelos de vision-lenguaje entrenados con datos masivos de Internet se incorporan directamente en el control robotico de extremo a extremo, y dichos modelos se denominan modelos de vision-lenguaje-accion (VLAs).

Notas del instructor

RT-2: Modelos Vision-Lenguaje-Accion Transfieren Conocimiento Web al Control Robotico -- DeepMind

DeepMind explora como incorporar modelos de vision-lenguaje entrenados con datos masivos de Internet directamente en el control robotico de extremo a extremo para mejorar la generalizacion e implementar inferencia semantica inesperada. Proponen el ajuste fino conjunto de modelos de vision-lenguaje de vanguardia, incluyendo datos de trayectoria del robot y tareas masivas de vision-lenguaje de Internet. Para incorporar respuestas de lenguaje natural y acciones del robot en el mismo formato, representan las acciones como etiquetas de texto y las agregan directamente al conjunto de entrenamiento del modelo. Tal modelo se llama Modelo de Vision-Lenguaje-Accion (VLA) e instanciado como RT-2.

El Poder de la IA en la Ciencia

El 8 de octubre de 2024, la Real Academia Sueca de Ciencias anuncio que el Premio Nobel de Fisica 2024 seria otorgado a John J. Hopfield y Geoffrey E. Hinton por sus "descubrimientos e invenciones fundamentales que permiten el aprendizaje automatico con redes neuronales artificiales".

El 9 de octubre de 2024, la Real Academia Sueca de Ciencias anuncio que el Premio Nobel de Quimica 2024 seria otorgado a David Baker por sus contribuciones al diseno computacional de proteinas, y a Demis Hassabis y John M. Jumper por sus contribuciones a la prediccion de la estructura de proteinas usando AlphaFold2, un modelo de IA desarrollado por ellos.

Notas del instructor

  • El Premio Nobel de Fisica fue otorgado al cientifico estadounidense John Hopfield y al cientifico canadiense Geoffrey Hinton por sus "descubrimientos e invenciones fundamentales que permiten el aprendizaje automatico con redes neuronales artificiales". El trabajo de estos dos pioneros en IA ha sentado las bases para el desarrollo del aprendizaje profundo y las redes neuronales.
  • La mitad del Premio Nobel de Quimica fue otorgada al cientifico estadounidense David Baker por su contribucion al diseno computacional de proteinas, mientras que el cientifico britanico Demis Hassabis y el cientifico estadounidense John Jumper compartieron la otra mitad por sus logros en prediccion de estructura de proteinas. AlphaFold2 ha resuelto un problema de 50 anos al predecir las estructuras complejas de aproximadamente 200 millones de proteinas conocidas. Ha sido utilizado por mas de 2 millones de personas en todo el mundo y tiene un significado revolucionario en campos como la I+D farmaceutica.
  • La concesion de estos premios refleja una nueva tendencia en la investigacion cientifica: la IA, como una fuerza que no puede ser ignorada, esta impulsando un cambio de paradigma en la investigacion cientifica. Ha establecido un paradigma teoricamente capaz de resolver todos los problemas cientificos: partiendo de problemas practicos, transformandolos en datos de entrada que la IA puede procesar, y luego usando redes de aprendizaje profundo para producir resultados.

Diapositiva 468

IA + Mecanica Cuantica

FermiNet (DeepMind, 2024)

FermiNet es una arquitectura de red neuronal propuesta por DeepMind. Soporta un metodo de representacion de funcion de onda parametrizada y puede procesar eficientemente sistemas cuanticos multielectronicos, mejorando la precision y eficiencia de los calculos de estructura electronica. FermiNet calcula la energia de atomos y moleculas, y aun captura el 97% de la energia relevante en un sistema molecular que consiste de 30 electrones de Biciclobutano.

David Pfau, et al. Accurate computation of quantum excited states with neural networks. Science, 2024

LapNet (Universidad de Pekin y ByteDance, 2024)

LapNet es una arquitectura de aprendizaje profundo desarrollada por ByteDance y la Universidad de Pekin para Monte Carlo variacional basado en redes neuronales (NN-VMC). El equipo ha disenado un marco computacional de Laplaciano hacia adelante para calcular el Laplaciano relacionado con la red neuronal a traves de un proceso efectivo de propagacion hacia adelante (que es un cuello de botella de NN-VMC). LapNet puede lograr una aceleracion significativa, extendiendo la aplicabilidad de NN-VMC a sistemas mas grandes.

Ruichen Li, et al. A computational framework for neural network based variational Monte Carlo with Forward Laplacian. Nature M. I., 2024

Diapositiva 469

IA + Ciencia de Materiales

GNoME (DeepMind y Nature, 2023)

La herramienta GNoME lanzada por DeepMind ha descubierto 2.2 millones de tipos de nuevos cristales, entre los cuales 380,000 cristales con estructuras estables se espera que sean sintetizados. Esto puede impulsar transformaciones tecnologicas en baterias de proxima generacion y superconductores. GNoME utiliza un modelo de red neuronal de grafos (GNN) para predecir eficientemente la estructura cristalina y estabilidad de los materiales, acelerando enormemente el descubrimiento de nuevos materiales. Comparado con los metodos experimentales tradicionales, los modelos de IA pueden predecir millones de materiales en poco tiempo, acortando el periodo de investigacion y ahorrando significativamente los costos experimentales.

Amil Merchant et al. Scaling deep learning for materials discovery, Nature (2023)

MatterGen (Microsoft, 2024)

MatterGen es un nuevo modelo generativo desarrollado por Microsoft Research para disenar materiales inorganicos con propiedades especificas. MatterGen puede generar estructuras cristalinas estables y diversas, y refinar tipos atomicos, coordenadas y redes periodicas a traves de un nuevo proceso de generacion basado en difusion. El modelo puede ser ajustado finamente segun requisitos de propiedades especificas para generar nuevos materiales que cumplan multiples restricciones de propiedades, como composicion quimica, simetria, y propiedades mecanicas, electronicas y magneticas, para el diseno direccional de materiales.

Tian Xie et al. MatterGen: a generative model for inorganic materials design. (2024)

IA + Dinamica Molecular

DeePMD (DP Technology, 2023)

DeePMD-kit es un paquete de aprendizaje profundo con gran potencial desarrollado por DP Technology. Integra modelado de primeros principios, metodos basados en datos representados por aprendizaje automatico y computacion de alto rendimiento para procesar problemas de dinamica molecular de escala ultra grande con la precision de primeros principios. DeePMD-kit puede mejorar significativamente la eficiencia de computo mientras mejora la precision de la simulacion, resolviendo la contradiccion entre precision y volumen de computo en los metodos de simulacion tradicionales.

Weinan E, et al. DeePMD-kit v2: A software package for deep potential models. 2023
