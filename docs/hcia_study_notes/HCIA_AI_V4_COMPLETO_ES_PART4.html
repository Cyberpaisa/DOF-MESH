Diapositiva 286

MoE (Mezcla de Expertos)

  • Mezcla de Expertos (MoE) es una estrategia de diseño de modelos que combina múltiples modelos ("expertos") para lograr un mejor rendimiento de predicción.
  • MoE puede mejorar efectivamente la capacidad y eficiencia de un modelo fundacional. Generalmente, MoE utiliza un mecanismo de compuerta junto con un conjunto de mecanismos de salida para combinar y equilibrar la selección de expertos, determinando finalmente la predicción final para cada experto. El mecanismo de selección de expertos elige expertos específicos para la predicción según la salida del mecanismo de compuerta.

Diapositiva 287

MoE — Notas del Instructor

Notas del instructor:
  • El tamaño del modelo es uno de los factores clave para mejorar el rendimiento del modelo. Por eso los modelos fundacionales actuales son tan efectivos. Con un presupuesto limitado de recursos computacionales, entrenar un modelo más grande con menos pasos de entrenamiento suele ser mejor que entrenar un modelo más pequeño con más pasos.
  • Una ventaja significativa de MoE es que se puede realizar un pre-entrenamiento efectivo con muchos menos recursos computacionales de los que requiere el modelo Denso. Esto significa que se puede escalar significativamente el modelo o el conjunto de datos bajo el mismo presupuesto computacional. Especialmente en la fase de pre-entrenamiento, el modelo de mezcla de expertos puede alcanzar el mismo nivel de calidad más rápidamente que el modelo Denso.

Diapositiva 311

PyTorch

  • PyTorch es un framework de computación de aprendizaje automático lanzado por Meta. Fue desarrollado basándose en Torch, un framework de computación científica que soporta muchos algoritmos de aprendizaje automático. Torch es una biblioteca de operaciones con tensores similar a NumPy, altamente flexible. Sin embargo, NumPy es menos popular porque usa el nicho lenguaje de programación Lua. Por eso se desarrolló PyTorch.
  • PyTorch es una biblioteca de tensores optimizada para aprendizaje profundo usando GPUs y NPUs. Es uno de los frameworks de aprendizaje profundo más comunes.

Diapositiva 312

Características de PyTorch

  • Python primero: PyTorch no simplemente vincula Python al framework C++. En su lugar, soporta directamente el acceso a Python a nivel granular. Los desarrolladores pueden usar PyTorch tan fácilmente como usar NumPy o SciPy.
  • Red neuronal dinámica: Por defecto, PyTorch usa grafos computacionales dinámicos. Los programas pueden construir o ajustar dinámicamente los grafos computacionales durante la ejecución.
  • Fácil depuración: PyTorch puede generar grafos dinámicos durante la ejecución, y los desarrolladores pueden detener el intérprete en el depurador y ver la salida de un nodo específico.
  • Ecosistema rico: PyTorch tiene una comunidad activa con muchos documentos, tutoriales, modelos y herramientas.

Diapositiva 313

TensorFlow

  • TensorFlow es una plataforma de aprendizaje automático de código abierto de extremo a extremo diseñada por Google. Actualmente, la versión principal es TensorFlow 2.X.
  • Proporciona múltiples niveles de APIs. Por ejemplo, se pueden usar APIs de alto nivel de Keras para construir y entrenar modelos rápidamente.
  • Para tareas de entrenamiento de aprendizaje profundo grandes, se puede usar la API de Estrategia de Distribución (para entrenamiento distribuido entre unidades de procesamiento) para realizar entrenamiento distribuido en diferentes configuraciones de hardware sin cambiar la definición del modelo.

Diapositiva 314

Herramientas de Producción de TensorFlow

  • TensorFlow.js: una biblioteca para desarrollar y entrenar modelos de aprendizaje automático en JavaScript. Puede desplegarse en navegadores o Node.js.
  • TensorFlow Lite: un framework de aprendizaje profundo de código abierto para inferencia en dispositivo. Se puede usar para desplegar modelos de aprendizaje automático en dispositivos móviles e IoT.
  • TensorFlow Extended (TFX): una plataforma de extremo a extremo para desplegar pipelines de aprendizaje automático en producción. Cuando estés listo para mover modelos entrenados de investigación a producción, usa TFX para crear y gestionar un pipeline de producción.

Diapositiva 315

JAX

  • JAX es una biblioteca de computación numérica de alto rendimiento desarrollada por Google. Combina APIs similares a NumPy, la función de diferenciación automática de Autograd y el compilador XLA de TensorFlow, implementando computación eficiente a través de CPUs, GPUs y TPUs.
  • Comparado con TensorFlow 2.x, proporciona:
    • APIs de programación funcional
    • Mayor eficiencia computacional

Formato de Datos Después de la Carga

  • Un conjunto de datos cargado generalmente toma la forma de un tensor multidimensional.
  • Datos de secuencia: usualmente un tensor 3D en formato [batch_size, sequence_length, feature_length]. Por ejemplo, embeddings de oraciones en NLP se expresan como [tamaño de lote, longitud de oración, dimensión de embedding de palabra].
  • Datos de imagen: usualmente un tensor 4D en formato [N,H,W,C]. N = tamaño del lote, H y W = altura y ancho en píxeles, C = número de canales.
  • En PyTorch, el formato predeterminado de tensores de datos de imagen recibidos es [N,C,H,W].
  • En TensorFlow, el formato predeterminado es [N,H,W,C].

Diapositiva 326

Grafo Computacional

  • Un grafo computacional proporciona una forma intuitiva de describir el proceso de computación en un modelo de aprendizaje profundo. Cada nodo representa una operación de computación, por ejemplo, suma, multiplicación y función de activación. Una arista representa un flujo de datos entre estas operaciones. Este método de representación estructurado hace que la computación compleja sea fácil de entender y analizar.
  • Realiza rápidamente la diferenciación automática.
  • Ayuda a encontrar el paralelismo de la red y los puntos de optimización.

Diapositiva 327

Composición del Grafo Computacional

  • Un grafo computacional contiene tensores (estructura de datos básica) y operadores (unidades de operación básicas).
  • Los nodos operadores en un grafo están conectados con aristas dirigidas, que indican el estado de cada tensor y las dependencias entre las operaciones.

Ejemplo: Tensor X → Matmul → Sigmoid → Tensor Y (con Tensor W como entrada a Matmul)

Notas del instructor:
  • Un nodo representa una variable u operación. Los nodos de entrada representan datos de entrada o parámetros, y los nodos intermedios y de salida representan el resultado del cálculo.
  • Los nodos están conectados con aristas, indicando el flujo de datos o dependencias computacionales.

Diapositiva 331

Grafo Computacional Dinámico

Un grafo computacional dinámico se construye durante la ejecución.

Ventajas:

  • Ejecución instantánea: Las operaciones se computan en tiempo real. La estructura del grafo se puede ajustar en cualquier momento.
  • Flexibilidad: Fácil de procesar diversos tamaños de datos. Soporta operaciones de flujo de control complejas.
  • Fácil depuración: Los desarrolladores pueden usar fácilmente herramientas de depuración de lenguajes como Python para verificar errores.
  • Amigable para investigación: Los investigadores pueden probar diferentes ideas e iterar modelos rápidamente.
  • Intuitividad: Los grafos dinámicos son más fáciles de entender y usar para principiantes.

Desventajas:

  • El grafo puede causar sobrecarga extra durante la ejecución, resultando en menor rendimiento que los grafos estáticos.
  • No se pueden optimizar tan exhaustivamente como los grafos estáticos.

Diapositiva 332

Uso de Grafos Dinámicos en TensorFlow

import tensorflow as tf
import time

# Inicialización de datos
x = tf.Variable(tf.random.normal([512, 1024])) # Tamaño de lote 512
W = tf.Variable(tf.random.normal([1024, 1024]))

# Configuración de prueba de rendimiento
iterations = 1000

# Medición de tiempo de ejecución
start_time = time.perf_counter()
for _ in range(iterations):
    _ = tf.matmul(x, W)  # Modo de ejecución inmediata
dynamic_duration = time.perf_counter() - start_time

print(f"[Grafo Dinámico] {iterations} iteraciones tomaron {dynamic_duration:.4f} segundos")

# [Grafo Dinámico] 1000 iteraciones tomaron 1.2873 segundos

Diapositiva 333

Grafo Computacional Estático

Un grafo computacional estático se define antes de la ejecución.

Ventajas:

  • Predefinición: El grafo computacional completo necesita definirse antes de la ejecución.
  • Optimización: Como la estructura del grafo es fija, el framework puede optimizar aún más, como asignación de memoria y computación paralela.
  • Amigable para despliegue: Los grafos estáticos son adecuados para entornos de producción y dispositivos móviles.
  • Portabilidad: Los grafos estáticos pueden exportarse en un formato independiente del framework.

Desventajas:

  • Requieren codificación más compleja, especialmente con flujos de control complejos.
  • La depuración puede ser menos intuitiva que los grafos dinámicos.

Diapositiva 334

Uso de Grafos Estáticos en TensorFlow

import tensorflow.compat.v1 as tf
import time
tf.disable_v2_behavior()
# Construcción del grafo computacional
x = tf.placeholder(tf.float32, shape=[None, 1024], name='input')
W = tf.Variable(tf.random_normal([1024, 1024]), name='weights')
y = tf.matmul(x, W) # Multiplicación de matrices

# Configuración de prueba de rendimiento
iterations = 1000
input_data = tf.random.normal([512, 1024])  # Tamaño de lote 512

# Medición de tiempo de ejecución
with tf.Session() as sess:
    sess.run(tf.global_variables_initializer())
    start_time = time.perf_counter()
    for _ in range(iterations):
        sess.run(y, feed_dict={x: input_data.eval()})
    static_duration = time.perf_counter() - start_time

print(f"[Grafo Estático] {iterations} iteraciones tomaron {static_duration:.4f} segundos")

# [Grafo Estático] 1000 iteraciones tomaron 0.3214 segundos

Computación Hacia Adelante

class MLP(nn.Module):
    # Declarar una capa con parámetros del modelo.
    def __init__(self, **kwargs):
        ...

    # Computación hacia adelante del modelo, es decir,
    # cómo computar y devolver la salida requerida basada en la entrada x.
    def forward(self, x):
        o = self.act(self.hidden(x))
        return self.output(o)

No es necesario definir una función de retropropagación. El sistema genera automáticamente la función backward requerida para la retropropagación calculando automáticamente el gradiente.

Diapositiva 352

Construcción de una Red Neuronal Completa (1)

Usar PyTorch para construir una red completa LeNet y usar MNIST para entrenamiento.

  • Construir una red LeNet:
    • Tiene dos capas convolucionales. La primera usa seis kernels de convolución 5x5 con paso de 1 y relleno de 2, y produce seis mapas de características. La segunda usa 16 kernels 5x5 con paso de 1 y sin relleno, produciendo 16 mapas de características.
    • La capa de agrupamiento promedio se usa para reducir las dimensiones de los mapas de características.
    • ReLU se usa después de las capas convolucionales, y sigmoid o softmax (en la capa de salida) se usa después de la capa completamente conectada.

Diapositiva 353

Construcción de una Red Neuronal Completa (2) — Definición de LeNet

import torch
import torch.nn as nn
import torch.optim as optim
import torchvision
import torchvision.transforms as transforms

# Definir la estructura de red LeNet.
class LeNet(nn.Module):
    def __init__(self):
        super(LeNet, self).__init__()
        # Definir una capa convolucional bidimensional.
        self.conv1 = nn.Conv2d(1, 6, kernel_size=5, stride=1, padding=2)
        self.conv2 = nn.Conv2d(6, 16, kernel_size=5, stride=1)
        # Definir la capa completamente conectada.
        self.fc1 = nn.Linear(16*5*5, 120)
        self.fc2 = nn.Linear(120, 84)
        self.fc3 = nn.Linear(84, 10)
        # Definir la capa de pooling.
        self.pool = nn.AvgPool2d(kernel_size=2, stride=2)
        # Función de activación
        self.relu = nn.ReLU()
        self.sigmoid = nn.Sigmoid()

Arquitectura de LeNet: Conv1 (6 kernels 5x5) → Pool (2x2) → Conv2 (16 kernels 5x5) → Pool → FC1 (120) → FC2 (84) → FC3 (10 salidas)

Diapositiva 354

Construcción de una Red Neuronal Completa (3) — Computación Hacia Adelante de LeNet

# Definir la estructura de red LeNet.
class LeNet(nn.Module):
    def __init__(self):
        ...

    def forward(self, x):
        x = self.conv1(x)
        x = self.relu(x)
        x = self.pool(x)
        x = self.conv2(x)
        x = self.sigmoid(x)
        x = self.pool(x)
        x = x.view(-1, 16*5*5)
        x = self.fc1(x)
        x = self.relu(x)
        x = self.fc2(x)
        x = self.relu(x)
        x = self.fc3(x)
        return x

# Entrenar la red.
for epoch in range(10):  # Entrenar 10 épocas.
    running_loss = 0.0
    for i, data in enumerate(trainloader, 0):
        inputs, labels = data
        optimizer.zero_grad()
        outputs = net(inputs)
        loss = criterion(outputs, labels)
        loss.backward()
        optimizer.step()

        running_loss += loss.item()
        if i % 100 == 99:  # Imprimir la pérdida cada 100 lotes.
            print(f'[Epoca {epoch+1}, Lote {i+1}] perdida: {running_loss/100:.3f}')
            running_loss = 0.0

Diapositiva 369

Compresión del Modelo

Aumentar el número de parámetros del modelo tiene el siguiente impacto:

Rendimiento de inferencia:

  • Cuando el ancho de banda de acceso a memoria del hardware es limitado, la sobrecarga de transferencia de datos aumenta con el incremento del peso.
  • Cuando no hay suficiente capacidad de VRAM, el modelo solo puede desplegarse en la CPU, reduciendo el rendimiento de inferencia.

Costo de despliegue:

  • Un peso mayor requiere más capacidad de almacenamiento, sobrecarga de recursos computacionales y capacidad de VRAM.

Actualmente, los dispositivos con gran VRAM y alto ancho de banda de acceso a memoria son costosos. Por lo tanto, la compresión del modelo es esencial para desplegar y promover aplicaciones de IA en dispositivos de consumo.

Notas del instructor:
  • A medida que el tamaño del lote y la longitud de la secuencia aumentan, la caché KV también aumenta, causando gran presión en el acceso a memoria.

Diapositiva 370

Método de Compresión — Poda del Modelo

La poda es una tecnología importante de compresión de modelos. La idea básica es eliminar pesos y ramas menos importantes de un modelo y esparcir la estructura de la red para obtener un modelo con menos parámetros. Sin embargo, la poda también puede causar deterioro del rendimiento del modelo. Por lo tanto, se debe encontrar un equilibrio entre el tamaño del modelo y el rendimiento.

Las conexiones neuronales en una red neuronal se representan matemáticamente como una matriz de pesos. Esto significa que la poda cambia algunos elementos en la matriz de pesos a elementos cero. El propósito es eliminar sinapsis o neuronas menos importantes.

Diapositiva 371

Método de Compresión — Destilación de Conocimiento

  • La destilación de conocimiento es una tecnología de compresión de modelos basada en aprendizaje por transferencia.
  • Una red grande y compleja generalmente proporciona buen rendimiento, pero también contiene mucha información redundante, lo que aumenta la complejidad computacional y el consumo de recursos.
  • La destilación extrae información útil de una red compleja y la migra a una red más pequeña. Después del aprendizaje, la red pequeña puede ofrecer un rendimiento similar a la red grande mientras requiere menos recursos computacionales.
  • La red compleja puede verse como un profesor, y la red pequeña como un estudiante.
