Diapositiva 76

Privacidad y Seguridad de Datos

El big data y la IA nos permiten obtener informacion de manera mas conveniente y eficiente. Sin embargo, nuestro comportamiento tambien se registra y los datos se usan todo el tiempo. ¿Como podemos proteger los datos privados?

  1. Usuarios: Antes de obtener la informacion del usuario, las aplicaciones moviles deben obtener el consentimiento del usuario (politica de privacidad). Los usuarios pueden negarse a proporcionar la informacion requerida por una aplicacion para proteccion proactiva.
  2. Sociedad: Promover activamente la formulacion de clausulas de proteccion de privacidad, implementar reglas basicas de la industria y fortalecer la supervision.
  3. Desarrolladores: Pueden usarse computacion confidencial, proteccion de privacidad de modelos, aprendizaje federado o aprendizaje adversario para prevenir ataques inversos y proteger la privacidad del usuario. Alternativamente, un marco de proteccion de privacidad bien establecido.
Notas del Instructor
  • Computacion confidencial: La transmision y computacion de datos son confidenciales; la proteccion de privacidad tiene alto costo.
  • Proteccion de privacidad de modelos: Los metodos comunmente usados incluyen anonimizacion de datos, entrenamiento con privacidad diferencial, cifrado de modelos y ofuscacion de modelos. (La privacidad diferencial puede medir y controlar la filtracion de datos entrenados por los modelos.)
  • Aprendizaje federado: Generalmente, se realiza aprendizaje automatico multi-parte sin compartir datos. Es esencialmente un marco de aprendizaje automatico distribuido con acceso limitado a datos. Los nodos de computo comparten solo gradientes, no datos sin procesar.

Diapositiva 94

Algoritmos de Aprendizaje Automatico (1)

El aprendizaje automatico se combina frecuentemente con metodos de aprendizaje profundo para estudiar y observar algoritmos de IA. Se dice que un programa de computadora aprende de la experiencia E con respecto a alguna clase de tareas T y medida de rendimiento P, si su rendimiento en las tareas en T, medido por P, mejora con la experiencia E.

  • Datos (Experiencia E): Informacion de la que el modelo aprende
  • Algoritmo de aprendizaje (Tarea T): Como el sistema de aprendizaje automatico debe procesar una muestra
  • Comprension (Medida de rendimiento P): Se usa para evaluar aspectos del aprendizaje automatico como la precision y la tasa de error
Notas del Instructor
  • El aprendizaje profundo es un subcampo del aprendizaje automatico. Para entender el aprendizaje profundo, primero debes comprender los fundamentos del aprendizaje automatico.
  • Experiencia E: La mayoria de los algoritmos de aprendizaje pueden entenderse como autorizados a experimentar un conjunto de datos completo. Algunos algoritmos no solo experimentan un conjunto de datos fijo. Por ejemplo, los algoritmos de aprendizaje por refuerzo interactuan con un entorno, por lo que hay un ciclo de retroalimentacion entre el sistema de aprendizaje y su proceso de entrenamiento. Los algoritmos de aprendizaje automatico pueden clasificarse ampliamente como no supervisados o supervisados segun el tipo de experiencia que les esta permitida tener.
  • Ejemplo - Aprender el juego de Go:
    • Experiencia E1: Juego consigo mismo — Aprendizaje no supervisado e indirecto
    • Experiencia E2: Consultar humanos durante un juego consigo mismo — Aprendizaje semi-supervisado
    • Experiencia E3: Aprender de juegos historicos humanos — Aprendizaje supervisado y directo
  • Ejemplo - Reconocimiento de escritura: Tarea T: reconocer texto manuscrito; Medida de rendimiento P: precision de clasificacion; Experiencia E: biblioteca de muestras clasificadas (aprendizaje supervisado y directo).

Diapositiva 97

Cuando Usar Aprendizaje Automatico (1)

El aprendizaje automatico proporciona soluciones a problemas complejos, o aquellos que involucran una gran cantidad de datos cuya funcion de distribucion no se puede determinar.

Considera los siguientes escenarios:

  • Las reglas son complejas o dificiles de describir, por ejemplo, reconocimiento de voz.
  • Las reglas de la tarea cambian con el tiempo, por ejemplo, etiquetado de partes del discurso, en el que se pueden generar nuevas palabras o significados en cualquier momento.
  • La distribucion de datos cambia con el tiempo y los programas necesitan adaptarse a nuevos datos constantemente, por ejemplo, pronostico de tendencias de ventas.

Diapositiva 99

Fundamento de los Algoritmos de Aprendizaje Automatico

  • Ecuacion objetivo (ideal): f: X → Y
  • Datos de entrenamiento: D: {(x1, y1), ..., (xn, yn)}
  • Algoritmo de aprendizaje produce una funcion hipotesis (real): g ≈ f

La funcion objetivo f es desconocida, y el algoritmo de aprendizaje no puede obtener una funcion f perfecta. La funcion hipotesis g aproxima la funcion f, pero puede diferir de ella.

Diapositiva 100

Principales Problemas Resueltos por el Aprendizaje Automatico

El aprendizaje automatico puede resolver muchos tipos de tareas. Los tres tipos mas comunes son:

  • Clasificacion: Especificar una de las k categorias para la entrada. El algoritmo de aprendizaje produce una funcion f: R^n → {1, 2, ..., k}. Ejemplo: algoritmos de clasificacion de imagenes en vision por computadora.
  • Regresion: El programa predice la salida para una entrada dada. El algoritmo produce una funcion f: R^n → R. Ejemplos: predecir el monto de reclamacion de un asegurado o predecir el precio de un valor.
  • Agrupamiento (Clustering): Basado en similitudes internas, el programa agrupa una gran cantidad de datos no etiquetados en multiples clases. Los datos de la misma clase son mas similares que los datos entre clases. Incluye busqueda por imagen y perfilamiento de usuarios.

La clasificacion y la regresion son dos tipos principales de tareas de prediccion. La salida de la clasificacion son valores de clase discretos, y la salida de la regresion son valores continuos.

Diapositiva 102

Tipos de Aprendizaje Automatico

  • Aprendizaje supervisado: El programa toma un conjunto conocido de muestras y entrena un modelo optimo para generar predicciones. Luego, el modelo entrenado mapea todas las entradas a salidas y realiza un juicio simple sobre las salidas. De esta manera, los datos desconocidos se clasifican.
  • Aprendizaje no supervisado: El programa construye un modelo basado en datos de entrada no etiquetados. Por ejemplo, un modelo de agrupamiento agrupa objetos basandose en similitudes.
  • Aprendizaje semi-supervisado: El programa entrena un modelo a traves de una combinacion de una pequena cantidad de datos etiquetados y una gran cantidad de datos no etiquetados.
  • Aprendizaje por refuerzo: Los sistemas de aprendizaje aprenden comportamiento del entorno para maximizar el valor de la funcion de senal de recompensa (refuerzo). En lugar de decirle al sistema la accion correcta, el entorno proporciona senales de refuerzo escalares para evaluar sus acciones.
  • Nuevos tipos: La evolucion del aprendizaje automatico esta produciendo nuevos tipos, por ejemplo, aprendizaje auto-supervisado, aprendizaje contrastivo, aprendizaje generativo.
Notas del Instructor
  • Aprendizaje supervisado: Le damos a una computadora un monton de preguntas de opcion multiple (muestras de entrenamiento) y proporcionamos respuestas estandar. La computadora intenta ajustar los parametros de su modelo para que sus predicciones se acerquen a las respuestas estandar.
  • Aprendizaje no supervisado: Le damos a una computadora preguntas de opcion multiple sin respuestas estandar. La computadora intenta analizar las relaciones entre estas preguntas y clasificarlas.
  • Aprendizaje semi-supervisado: El aprendizaje supervisado tradicional usa un gran numero de muestras etiquetadas de entrenamiento. Sin embargo, etiquetar datos requiere mucho trabajo y tiempo. Ejemplo: recomendacion de paginas web donde solo unos pocos usuarios estan dispuestos a etiquetar las que les gustan.
  • Aprendizaje por refuerzo: Le damos a una computadora preguntas sin respuestas. Ella intenta resolverlas, y nosotros, como maestros, verificamos si las respuestas son correctas. Si genera mas respuestas correctas, ofrecemos mas recompensas. No estrictamente hablando, el aprendizaje por refuerzo puede entenderse como aprendizaje no supervisado mas aprendizaje supervisado.

Diapositiva 113

Concepto Basico del Aprendizaje Automatico — Conjunto de Datos

Conjunto de datos: Coleccion de datos usados en tareas de aprendizaje automatico, donde cada pieza de datos se llama una muestra. Los elementos o atributos que reflejan la presentacion o naturaleza de una muestra en un aspecto particular se llaman caracteristicas.

  • Conjunto de entrenamiento: Conjunto de datos usado en el proceso de entrenamiento, donde cada muestra se llama una muestra de entrenamiento. El aprendizaje (o entrenamiento) es el proceso de construir un modelo a partir de datos.
  • Conjunto de prueba: Conjunto de datos usado en el proceso de prueba, donde cada muestra se llama una muestra de prueba. La prueba se refiere al proceso en el cual el modelo aprendido se usa para prediccion.

Diapositiva 115

Importancia del Procesamiento de Datos

Los datos son cruciales para los modelos y determinan el alcance de las capacidades del modelo. Todos los buenos modelos requieren buenos datos.

Preprocesamiento de datos:

  • Limpieza de datos: Rellenar valores faltantes, detectar y eliminar ruido y otros puntos anormales.
  • Estandarizacion de datos: Estandarizar datos para reducir ruido y mejorar la precision del modelo.
  • Reduccion de dimensionalidad de datos: Simplificar atributos de datos para evitar la maldicion de la dimensionalidad.

Diapositiva 117

Limpieza de Datos

La mayoria de los modelos de aprendizaje automatico procesan caracteristicas, que son generalmente representaciones numericas de variables de entrada que pueden usarse en el modelo.

En la mayoria de los casos, solo los datos preprocesados pueden ser usados por los algoritmos. El preprocesamiento de datos involucra las siguientes operaciones:

  • Filtrado de datos
  • Manejo de datos perdidos
  • Manejo de posibles valores erroneos o anormales
  • Fusion de datos de multiples fuentes
  • Consolidacion de datos

Diapositiva 118

Datos Sucios

Los datos sin procesar generalmente contienen problemas de calidad de datos:

  • Incompletitud: Datos incompletos o falta de atributos o valores relevantes.
  • Ruido: Los datos contienen registros incorrectos o puntos anormales.
  • Inconsistencia: Los datos contienen registros conflictivos.

Tipos de problemas: Valores faltantes, valores invalidos, valores en campos incorrectos, elementos duplicados invalidos, formato incorrecto, atributos dependientes, errores ortograficos.

Diapositiva 119

Conversion de Datos

Los datos preprocesados necesitan convertirse a una representacion adecuada para modelos de aprendizaje automatico. Se usan tipicamente los siguientes metodos:

  • Codificar datos categoricos en numeros para clasificacion
  • Convertir datos numericos en datos categoricos para reducir los valores de las variables (por ejemplo, segmentar datos de edad)
  • Otros datos:
    • Embedding de palabras en texto para convertirlas en vectores de palabras (tipicamente se usan modelos como word2vec y BERT)
    • Procesamiento de datos de imagen: conversion de espacio de color, conversion a escala de grises, conversion geometrica, caracteristicas Haar-like y mejora de imagen
  • Ingenieria de caracteristicas:
    • Normalizar y estandarizar caracteristicas para asegurar que diferentes variables de entrada caigan en el mismo rango de valores
    • Aumento de caracteristicas: combinar o convertir variables existentes para generar nuevas caracteristicas, como promedios

Diapositiva 120

Necesidad de la Seleccion de Caracteristicas

Generalmente, un conjunto de datos tiene muchas caracteristicas, algunas de las cuales pueden ser innecesarias o irrelevantes para los valores a predecir.

La seleccion de caracteristicas es necesaria en los siguientes aspectos:

  • Simplifica modelos para facil interpretacion
  • Acorta el tiempo de entrenamiento
  • Evita la maldicion de la dimensionalidad
  • Mejora la generalizacion del modelo y evita el sobreajuste

Diapositiva 121

Metodos de Seleccion de Caracteristicas — Filtro (Filter)

Los metodos de filtro son independientes de los modelos durante la seleccion de caracteristicas. Al evaluar la correlacion entre cada caracteristica y el atributo objetivo, un metodo de filtro puntua cada caracteristica usando una medida estadistica y luego ordena las caracteristicas por puntaje.

Metodos comunes:

  • Coeficiente de correlacion de Pearson
  • Coeficiente chi-cuadrado
  • Informacion mutua

Limitaciones: Los metodos de filtro tienden a seleccionar variables redundantes porque no consideran las relaciones entre caracteristicas.

Proceso: Recorrer todas las caracteristicas → Seleccionar el mejor subconjunto de caracteristicas → Algoritmo de aprendizaje → Evaluacion del modelo.

Diapositiva 122

Metodos de Seleccion de Caracteristicas — Envolvente (Wrapper)

Los metodos envolventes usan un modelo de prediccion para puntuar un subconjunto de caracteristicas. Tratan la seleccion de caracteristicas como un problema de busqueda y evaluan y comparan diferentes combinaciones.

Metodo comun: Eliminacion recursiva de caracteristicas.

Limitaciones:

  • Entrenan un nuevo modelo para cada subconjunto de caracteristicas, lo que puede ser computacionalmente intensivo.
  • Generalmente proporcionan conjuntos de caracteristicas de alto rendimiento para un tipo especifico de modelo.

Proceso: Recorrer todas las caracteristicas → Generar subconjunto de caracteristicas → Algoritmo de aprendizaje → Evaluacion del modelo → Seleccionar el mejor subconjunto.

Diapositiva 123

Metodos de Seleccion de Caracteristicas — Embebido (Embedded)

Los metodos embebidos tratan la seleccion de caracteristicas como parte del proceso de modelado.

La regularizacion es el tipo mas comun de metodos embebidos. Los metodos de regularizacion, tambien llamados metodos de penalizacion, introducen restricciones adicionales en la optimizacion de un algoritmo predictivo para sesgar el modelo hacia menor complejidad y reducir el numero de caracteristicas.

Metodo comun: Regresion LASSO.

Proceso: Recorrer todas las caracteristicas → Generar subconjunto de caracteristicas → Algoritmo de aprendizaje + Evaluacion del modelo → Seleccionar el subconjunto mas apropiado.

Notas del Instructor

Se destaca el atributo de penalizacion del modelo, que se usa para eliminar caracteristicas insignificantes.

  1. Dividir datos: Dividir el conjunto de datos en conjuntos de entrenamiento, validacion y prueba.
  2. Entrenar el modelo: Usar datos procesados por limpieza de datos e ingenieria de caracteristicas.
  3. Validar el modelo: Usar el conjunto de validacion para evaluar la efectividad del modelo.

Diapositiva 127

¿Que es un Buen Modelo?

  • Generalizacion: La precision de las predicciones basadas en datos reales.
  • Explicabilidad: Los resultados predichos son faciles de explicar.
  • Velocidad de prediccion: El tiempo necesario para hacer una prediccion.
Notas del Instructor

La capacidad de generalizacion es lo mas importante.

Diapositiva 128

Efectividad del Modelo (1)

  • Capacidad de generalizacion: El aprendizaje automatico busca asegurar que los modelos funcionen bien en nuevas muestras, no solo en las usadas para entrenamiento. La capacidad de generalizacion, tambien llamada robustez, es el grado en que un modelo aprendido puede aplicarse a nuevas muestras.
  • Error: La diferencia entre la prediccion de un modelo aprendido sobre una muestra y el resultado real de la muestra.
    • Error de entrenamiento: Error del modelo en el conjunto de entrenamiento.
    • Error de generalizacion: Error del modelo en nuevas muestras. Preferimos un modelo con un error de generalizacion mas pequeno.
  • Subajuste (Underfitting): El error de entrenamiento es grande.
  • Sobreajuste (Overfitting): El error de entrenamiento del modelo entrenado es pequeno mientras que el error de generalizacion es grande.
Notas del Instructor
  • Sobreajuste: Ocurre frecuentemente en modelos matematicos complejos. Para prevenir el sobreajuste, podemos simplificar modelos matematicos, terminar el entrenamiento antes del sobreajuste, o usar metodos de dropout/decaimiento de peso.
  • Subajuste: Ocurre si el modelo matematico es demasiado simple o el tiempo de entrenamiento es muy corto. Para resolver el problema de subajuste, usa un modelo mas complejo o extiende el tiempo de entrenamiento.

Diapositiva 129

Efectividad del Modelo (2)

Capacidad del modelo, tambien conocida como complejidad del modelo, es la capacidad del modelo para ajustar diversas funciones.

  • Con suficiente capacidad para manejar la complejidad de la tarea y los volumenes de datos de entrenamiento, los resultados del algoritmo son optimos.
  • Los modelos con capacidad insuficiente no pueden manejar tareas complejas porque puede ocurrir subajuste.
  • Los modelos con gran capacidad pueden manejar tareas complejas, pero puede ocurrir sobreajuste cuando la capacidad es mayor que la cantidad requerida por una tarea.

Ilustracion: Subajuste (caracteristicas no aprendidas) → Buen ajuste → Sobreajuste (ruido aprendido)

Notas del Instructor

La capacidad efectiva esta restringida por algoritmos, parametros y regularizacion.

Diapositiva 130

Causa del Sobreajuste — Errores

Error de prediccion = Sesgo^2 + Varianza + Error ineliminable

En general, los dos factores principales del error de prediccion son la varianza y el sesgo.

  • Varianza: Cuanto se desvia un resultado de prediccion de la media. La varianza es causada por la sensibilidad del modelo a pequenas fluctuaciones en un conjunto de entrenamiento.
  • Sesgo: Diferencia entre el promedio de los valores predichos y los valores reales.
Notas del Instructor

Sobre el error ineliminable: Teoricamente, si hubiera una cantidad infinita de datos y un modelo perfecto, el error podria eliminarse. En la practica, ningun modelo es perfecto y nuestro volumen de datos es limitado.

Diapositiva 131

Varianza y Sesgo

Diferentes combinaciones de varianza y sesgo:

  • Bajo sesgo y baja varianza → Buen modelo
  • Bajo sesgo y alta varianza → Modelo inadecuado
  • Alto sesgo y baja varianza → Modelo inadecuado
  • Alto sesgo y alta varianza → Mal modelo

Un modelo ideal puede capturar con precision las reglas en los datos de entrenamiento y generalizarse a datos invisibles (nuevos). Sin embargo, es imposible que un modelo complete ambas tareas al mismo tiempo.

Diapositiva 132

Complejidad y Errores de los Modelos

  • Cuanto mas complejo es un modelo, menor es su error de entrenamiento.
  • A medida que la complejidad del modelo aumenta, el error de prueba disminuye antes de aumentar nuevamente, formando una curva convexa.

Grafica: Eje Y = Error, Eje X = Complejidad del modelo. El error de prueba forma una curva en U, mientras que el error de entrenamiento disminuye continuamente.

Diapositiva 133

Evaluacion de Rendimiento del Aprendizaje Automatico — Regresion

  • Error absoluto medio (MAE): Un valor de MAE mas cercano a 0 indica que el modelo se ajusta mejor a los datos de entrenamiento.
    MAE = (1/m) * SUM(|yi - mi|)
  • Error cuadratico medio (MSE):
    MSE = (1/m) * SUM((yi - mi)^2)
  • R^2 (Coeficiente de determinacion): El rango de valores de R^2 es [0, 1]. Un valor mayor indica que el modelo se ajusta mejor a los datos de entrenamiento. TSS indica la diferencia entre muestras, y RSS indica la diferencia entre los valores predichos y los valores de muestra.
    R^2 = 1 - RSS/TSS = 1 - SUM((yi - mi)^2) / SUM((yi - y_media)^2)
