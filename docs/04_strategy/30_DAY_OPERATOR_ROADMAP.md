# DOF-MESH: The 30-Day Operator Roadmap (2026 Strategy)

## 1. Vision: From User to Operator
The DOF-MESH project is transitioning from a "collection of scripts" to a **Sovereign Infrastructure**. This roadmap defines the curriculum to move from "Version 1" (manual prompting) to "Version 2" (infrastructure deployment).

## 2. Core Pillars

### 2.1. Context Engineering (The New Alpha)
* **Principle**: Clarity beats cleverness.
* **Skill**: Instead of individual "magic spell" prompts, lead with **structural context** (XML tags for Claude, JSON for Gemini).
* **Implementation**: Every agent in the Legion must strictly follow the `<context>`, `<task>`, `<format>` triad.

### 2.2. Model Specialization (Strategic Selection)
* **Claude (Anthropic)**: Coding, Marketing, Nuance, Spreadsheets.
* **Gemini (Google)**: Research, 1M+ token context window, current data (Google Search integration).
* **Ollama (Local)**: Sovereign inference, low cost, data privacy.
* **Nano Banana Pro**: Generation of accurate infographics and visuals with perfect text rendering.

### 2.3. Vibe Coding & Infrastructure
* **Tooling**: Cursor + Claude Code for 10x shipping speeds.
* **Automation**: n8n for persistent, "while-you-sleep" workflows.
* **Protocol**: Model Context Protocol (MCP) as the universal adapter for data sources.

## 3. The 30-Day Execution Cycle (DOF-MESH specific)
* **Phase 1 (Fundamentals)**: Master the attention mechanism; understand tokenization and context architecture.
* **Phase 2 (RAG & Knowledge)**: Deploy NotebookLM and Claude Projects; ground agents in the "Sovereign Vault".
* **Phase 3 (Creation)**: Use VEO/Kling for B-roll and Nano Banana for technical schematics.
* **Phase 4 (Automation)**: Connect the mesh via n8n/MCP. Deploy "The Citadel" (Docker).

## 4. The End State: Personal AI Agents
We are building toward a **Clawdbot-style architecture**:
* Runs on local hardware (Mac mini/VPS).
* Connects to all mesh platforms.
* Self-modifying extensions.
* Absolute data sovereignty.
