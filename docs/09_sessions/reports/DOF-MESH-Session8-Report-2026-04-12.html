03 · Features
Lo que se Construyó.
Parte 1 — Infraestructura Base (Fases 2 & 3)
Fase 2
DiskTaskQueue — Persistencia JSONL con filelock
core/disk_task_queue.py
Cola de tareas persistente en disco usando JSONL atómico con filelock. Soporte TTL por tarea, operaciones enqueue / dequeue / peek / drain. Worker-safe para procesos concurrentes. Integrada en el pipeline de AbortSignal y StreamingToolExecutor.
Fase 2
SystemPromptBoundary — Detección de injection/leakage
core/system_prompt_boundary.py
Capa de seguridad que detecta intentos de prompt injection y leakage de system prompt. Patrones regex compilados para 12+ vectores de ataque. Integrable en cualquier handler de input/output. Alimenta directamente el AutonomousDaemon governance gate.
Fase 3
AbortSignal Cascade — Cancelación en claude_commander
core/claude_commander.py
Sistema de cancelación en cascada que propaga señales de abort desde el commander hasta los subprocesos y tools activos. Permite terminar runs costosos sin dejar procesos huérfanos. Integrado con el event loop asíncrono del daemon.
Fase 3
StreamingToolExecutor — JSONL event stream + abort + retry backoff
core/streaming_tool_executor.py
Ejecutor de tools que emite eventos JSONL en streaming: tool_start, tool_result, tool_error, tool_abort. Backoff exponencial configurable en retries. Compatible con DiskTaskQueue para encolado de ejecuciones. Soporte para abort mid-execution via AbortSignal.
Fase 3
Git Worktree Isolation — Workers en /tmp
core/git_worktree_isolation.py
Workers spawneados en worktrees de Git aislados bajo /tmp/dof-worker-*. Cada worker opera en su propio árbol de trabajo sin contaminar el repo principal. Limpieza automática al finalizar. Integra con el script spawn_claude_worker.sh.
Parte 2 — Meta-Governance + Release v0.7.0
Governance
AutonomousDaemon Governance Gate
core/autonomous_daemon.py
Método _gate_instruction() añadido al daemon. Antes de ejecutar cualquier instrucción en el ciclo autónomo, pasa por SystemPromptBoundary. Instrucciones con injection detectada son bloqueadas con log de auditoría. Zero-LLM: determinístico por regex.
Meta-Gov
DOFFeatureFlags — 3-layer priority, 12 flags, Policy-as-Code
core/feature_flags.py
Sistema de feature flags con prioridad: env vars > flags.json > defaults hardcoded. 12 flags cubren: governance strictness, streaming, worktree isolation, semantic avoidance, GraphifyTool, experimental modules. Integrado con Constitution para gating de reglas. Pattern: flags.is_enabled("STRICT_GOVERNANCE").
Meta-Gov
DaemonMemory — Indexa cycles.jsonl, 553 ciclos reales
core/daemon_memory.py
Índice semántico sobre el historial real del daemon autónomo. Lee logs/daemon/cycles.jsonl, extrae métricas por ciclo (score, violations, actions), permite queries como "últimas 10 iteraciones con score > 0.8" o "ciclos donde se bloqueó por governance". Base para _apply_semantic_avoidance().
Release
MANIFEST.in + v0.7.0 bump + PyPI publish
MANIFEST.in · pyproject.toml · dof/__init__.py
MANIFEST.in con exclusiones explícitas: logs/, *.jsonl, *.key, *.pem, .env*. Version bump sincronizado en pyproject.toml y dof/__init__.py. dof-sdk==0.7.0 publicado en PyPI. GitHub Release v0.7.0 con changelog.
Parte 3 — Semántica + CI Fix
Governance
feature_flags_governance — Constitution rules gateadas por flags
core/governance.py · core/feature_flags.py
Integración entre FeatureFlags y Constitution. Las reglas de governance pueden activarse/desactivarse por flag sin modificar código. Ejemplo: STRICT_GOVERNANCE=false relaja las HARD_RULES a modo advertencia. Permite ajuste fino por entorno (dev/staging/prod).
CI Fix
CI fix — __version__ dinámico en tests
tests/test_version.py · core/__init__.py
Tests de versión que hardcodeaban "0.6.0" roto tras el bump a 0.7.0. Reemplazados con importación dinámica de dof.__version__. CI vuelve a verde. Patrón canónico para todos los tests de versión futuros.
Semántica
DaemonMemory Semantic Avoidance
core/autonomous_daemon.py · core/daemon_memory.py
Método _apply_semantic_avoidance() en el daemon. Antes de proponer una acción, consulta DaemonMemory para detectar patrones que históricamente terminaron en violación de governance. Si el score de similitud supera umbral, el daemon desvía la acción. Auto-mejora determinística sin LLM.
CrewAI
GraphifyTool — CrewAI tool con fallback AST
core/graphify_tool.py
Tool CrewAI que convierte el codebase en grafo de dependencias navegable. Fallback automático a análisis AST cuando el grafo no está disponible. Los agentes pueden preguntar: "¿qué módulos importa governance.py?" o "¿quién usa DiskTaskQueue?". Mejora razonamiento de agentes sobre arquitectura.
