# Skill: Knowledge Brain — Cerebro Evolutivo para AGI Soberana

## Cuándo usar este skill
Cuando el usuario comparte repos, tools, papers, tweets, screenshots, o cualquier información sobre el ecosistema AI/crypto/agents. Este skill absorbe, documenta, evalúa y distribuye el conocimiento.

## Flujo de Absorción

### 1. INGEST — Recibir y clasificar
- Identificar tipo: repo, tool, paper, benchmark, protocol, framework, service
- Extraer: nombre, URL, qué hace, tech stack, licencia
- Clasificar relevancia: ALTA / MEDIA / BAJA para el ecosistema DOF

### 2. EVALUATE — Evaluar utilidad
Para cada pieza de conocimiento, responder:
- ¿Nos sirve para DOF (governance, observabilidad, agentes)?
- ¿Nos sirve para Mission Control (dashboard, paneles, visualización)?
- ¿Nos sirve para los 14 agentes OpenClaw?
- ¿Nos sirve para la AGI soberana local (memoria, razonamiento, auto-evolución)?
- ¿Es gratuito o tiene free tier?
- ¿Es open source?
- ¿Se puede correr en M4 Max local?

### 3. DOCUMENT — Documentar en formato estándar
Agregar a `docs/TOOLS_AND_INTEGRATIONS.md` con formato:

```markdown
## N. [Nombre] — [Descripción corta]
- **URL**: [link]
- **Qué es**: [1-2 oraciones]
- **Install/Precio**: [cómo obtener]

### Features clave
[tabla o lista de capacidades principales]

### Relevancia para DOF
[cómo beneficia específicamente nuestro sistema]
- **Estado**: EVALUANDO / DISPONIBLE / IMPLEMENTAR / REFERENCIA
```

### 4. DISTRIBUTE — Distribuir a agentes
- Actualizar `agents/synthesis/SOUL.md` con resumen ejecutivo
- Si es muy relevante, agregar a tabla de Links Rápidos
- Si afecta roadmap, actualizar Phase 2/3

### 5. CONNECT — Conectar con conocimiento existente
- ¿Se conecta con algo ya documentado? (ej: ASMR + Ori Mnemos = memory stack)
- ¿Reemplaza algo existente? (ej: ASMR > ChromaDB)
- ¿Habilita algo nuevo? (ej: CryptoSkill → distribución de skills DOF)

## Principios del Cerebro
1. **Soberanía**: Preferir local sobre cloud, open source sobre propietario
2. **Determinismo**: Preferir verificable sobre probabilístico
3. **Evolución**: El conocimiento crece con uso, se poda por negligencia (Ebbinghaus)
4. **Conexión**: Cada pieza se conecta con el grafo existente
5. **Acción**: Documentar no es suficiente — evaluar → planificar → implementar
